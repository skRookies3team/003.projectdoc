## 회의 내용

### 안건 1: AWS Bedrock 연동 및 SSE 스트리밍 CORS 이슈 해결
* **논의 내용**: 생성형 AI 모델(Claude 3.5) 연동과 실시간 응답(SSE) 구현 중 발생한 프론트엔드 연결 문제 해결.
* **이슈 및 해결**:
    * **이슈**: Frontend에서 SSE 연결 시도 시 CORS 정책 위반으로 인한 실패 발생.
    * **해결**: `CorsConfig` 설정을 추가하여 CloudFront 도메인을 허용함으로써 정상적인 스트리밍 통신 성공.
* **진행 현황**:
    * **양승준**: AWS Bedrock Runtime SDK 추가 및 Claude 3.5 (Haiku/Sonnet) 모델 호출 성공.

### 안건 2: Kafka 및 Vector DB 로직 안정화
* **논의 내용**: 이전에 발생하던 Kafka 이벤트 발행 오류 수정 및 RAG를 위한 벡터 DB 저장 로직 완성.
* **결정 사항**:
    * **김지은**: 
        * 지속되던 Kafka 이벤트 발행 오류를 해결하고 다이어리 생성 시 비동기 이벤트 처리가 정상 동작함을 확인.
        * Milvus 연동 및 벡터 DB 저장 로직 구현을 완료함.

### 안건 3: 펫메이트 실시간 채팅 기능 구현
* **논의 내용**: 사용자 간 매칭 후 실시간 소통을 위한 WebSocket 기반 채팅 백엔드 구축.
* **결정 사항**:
    * **신재석, 김현근**: 펫메이트 서비스 내 매칭된 사용자 간의 채팅 기능을 위해 WebSocket 통신 프로토콜을 구현 중.

### 안건 4: 인프라 배포 파이프라인 및 도메인 설정
* **논의 내용**: 서비스 배포를 위한 CI/CD 설정 및 도메인 연결 작업.
* **결정 사항**:
    * **손민정**: 
        * `record-service`의 Dockerfile 및 Git Action 작성, Helm chart 작성 완료.
        * 프론트엔드 배포 방식을 S3 정적 호스팅으로 구성.
        * Route 53 도메인 구매 및 ALB(Application Load Balancer) 연결 작업 진행.

### 안건 5: 알림 및 소셜 서비스 기능 구현
* **논의 내용**: 사용자 알림 API 개발 및 소셜 서비스 프론트엔드 마감.
* **결정 사항**:
    * **이가은**: 알림 생성, 읽기 처리, 내역 조회 등 알림 관련 핵심 API 구현 진행.
    * **김현근**: 소셜 서비스의 UI 최종 마무리 작업 진행.
