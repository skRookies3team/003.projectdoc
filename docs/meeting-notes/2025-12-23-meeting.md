## 회의 안건
* **1**: Kafka 및 Vector DB(Milvus) 환경 구축 및 데이터 처리 로직 구현
* **2**: Diary-Healthcare 서비스 간 이벤트 기반 비동기 통신 아키텍처 설계
* **3**: 소셜 서비스 FE/BE 통합 연동 및 검색 기능 최적화
* **4**: AI 기반 동물 사진 분석 API 구현
* **5**: 인프라 CDN 통신 테스트 및 프론트엔드 배포 연동

## 회의 내용

### 안건 1: Kafka 및 Vector DB(Milvus) 환경 구축
* **논의 내용**: RAG(검색 증강 생성) 및 대용량 처리를 위한 핵심 인프라 셋팅 진행 상황 공유.
* **결정 사항**:
    * **김지은**: Milvus Vector DB 연동을 완료하고 벡터 데이터 저장 로직을 구현함.
    * Kafka 환경 셋팅을 마치고, 서비스 간 데이터 교환을 위한 이벤트 발행(Publish) 로직을 구현 완료함.

### 안건 2: Diary-Healthcare 서비스 간 비동기 통신 설계
* **논의 내용**: 일기(Diary) 데이터가 생성되었을 때 헬스케어(Healthcare) 서비스로 데이터를 전달하는 파이프라인 설계.
* **결정 사항**:
    * **양승준**: 서비스 간 결합도를 낮추기 위해 `Diary Service → Kafka → Healthcare Service`로 이어지는 비동기 이벤트 기반 아키텍처 설계를 확정함.

### 안건 3: 소셜 서비스 FE/BE 통합 및 검색 기능 수정
* **논의 내용**: 소셜 피드 기능의 프론트엔드 UI 연동과 백엔드 검색 API의 정확도 개선 작업.
* **결정 사항**:
    * **김현근**: 
        * [FE] 소셜 서비스 API 연동을 마치고 개인 프로필 페이지 UI/UX 수정 및 데이터 바인딩 완료.
        * [BE] 소셜 서비스 내 사용자/피드 검색 기능의 로직을 수정하여 결과 반환 오류 해결.

### 안건 4: AI 기반 동물 사진 분석 API 구현
* **논의 내용**: 업로드된 반려동물 사진을 분석하여 종(Species)이나 상태를 파악하는 기능 개발.
* **결정 사항**:
    * **이가은**: 동물 사진 분석을 위한 백엔드 API 구현을 진행함 (AI 모델 서빙 연동 포함).

### 안건 5: 인프라 통신 테스트 및 구성 보조
* **논의 내용**: 실제 배포 환경에서의 CDN 접속 경로 및 Pod 간 통신 상태 점검.
* **결정 사항**:
    * **손민정**: CDN에서 EKS Pod까지의 네트워크 트래픽 통신 테스트를 수행하고 프론트엔드 리소스 연동 상태를 확인함.
    * **신재석**: 인프라 구성 과정에서 발생하는 이슈 트러블슈팅 및 환경 설정 보조 수행.
