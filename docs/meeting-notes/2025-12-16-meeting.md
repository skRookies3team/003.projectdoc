## 회의 내용

### 안건 1: 금주 핵심 작업 할당 및 역할 분담 (To-Do 확정)
* **논의 내용**: 프로젝트 후반부 핵심 기능인 AI 서비스, 쇼핑/소셜 기능 구현 및 인프라 배포 자동화에 대한 역할을 분담함.
* **결정 사항**: 아래 표와 같이 작업을 할당하고 진행한다.

| 담당자 | 핵심 작업 | 비고 |
| :--- | :--- | :--- |
| **양승준** | 일반 챗봇 구현 (Claude 3 Haiku 연동), ChatLog DB 저장 로직 구현, SSE 엔드포인트 개설 | 챗봇 스트리밍 통신 담당 |
| **김지은** | 일기 생성 시 petId 반환 오류 수정 (Service 간 DTO 정합성), AI 일기 생성 기능 통합, social 로컬 테스트 | AI 서비스 로직 통합 |
| **이가은** | 소셜id 기반 사용자 검색 API 구현, 회원가입 오류 수정 및 기본 프로필 이미지 설정 로직 구현 | 회원 서비스 예외 처리 개선 |
| **신재석** | 쇼핑페이지 프론트-백 연결 및 API 생성, 펫메이트 위치 DB 저장 기능 설정 | 쇼핑 및 위치 기반 서비스 |
| **김현근** | 해시태그, 사용자 검색 및 인기 게시물 조회 기능 구현 | 소셜 서비스 검색 기능 고도화 |
| **손민정** | apigateway 및 user-service Helm Chart 작성, ArgoCD 배포 | 쿠버네티스 배포 자동화 |

### 안건 2: AI 기반 일기 생성 및 스트리밍 챗봇 구현 방안
* **논의 내용**: Bedrock Claude 3 모델을 활용한 스트리밍 챗봇 구현과 실시간 데이터 전송을 위한 SSE 기술 도입 논의.
* **결정 사항**:
    * Bedrock Claude 3 Haiku 모델을 연동하고, 대화 내용 저장을 위한 ChatLog 엔티티를 생성한다.
    * 서버와 클라이언트 간 실시간 통신을 위해 SSE(Server-Sent Events) 방식을 채택하여 ChatController를 개설한다.
    
* **담당자**: 양승준
* **마감일**: 2025-12-26

### 안건 3: 서비스 간 DTO 정합성 확보 및 회원가입 예외 처리 로직 개선
* **논의 내용**: 마이크로서비스 간 통신 시 발생하는 데이터 모델 불일치 문제 해결과 사용자 편의를 위한 기본값 설정 논의.
* **결정 사항**:
    * 유저 서비스와 다이어리 서비스 간의 DTO를 맞춰 petId 반환 오류를 해결하고, 프론트엔드 펫 선택 UI를 수정한다.
    * 회원가입 시 이미지를 업로드하지 않는 사용자를 위해 기본 프로필 이미지 자동 설정 로직을 추가한다.
* **담당자**: 김지은, 이가은
* **마감일**: 2025-12-26

### 안건 4: 쇼핑 및 소셜 서비스 백엔드 기능 고도화
* **논의 내용**: 검색 기능 강화 및 위치 기반 데이터 관리, 프론트-백 연동 최적화 논의.
* **결정 사항**:
    * 쇼핑 페이지의 API를 생성하고 프론트엔드와 연결하며, 펫메이트 서비스의 위치 정보 DB 저장 로직을 완성한다.
    * 소셜 서비스 내 해시태그 및 인기 게시물 필터링 로직을 구현하여 검색 품질을 높인다.
* **담당자**: 신재석, 김현근
* **마감일**: 2025-12-26

### 안건 5: Helm Chart를 활용한 인프라 배포 자동화
* **논의 내용**: ArgoCD를 활용한 선언적 배포를 위해 각 마이크로서비스의 설정을 패키징하는 방안 논의.
* **결정 사항**:
    * apigateway와 user-service에 대한 Helm Chart를 작성하여 배포 표준을 만들고, ArgoCD를 통해 쿠버네티스 클러스터에 배포한다.
    
* **담당자**: 손민정
* **마감일**: 2025-12-26
