### 안건 2: 위치 및 날씨 데이터 연동을 통한 다이어리 서비스 고도화
* **논의 내용**: 다이어리 생성 시 사용자의 현재 위치와 날씨 정보를 자동으로 기록하고, 지도를 통해 시각화하는 방안 논의.
* **결정 사항**:
    * 카카오맵 API를 사용하여 지도 UI를 구현하고, 기상청 API를 통해 실시간 날씨 정보를 받아온다.
    * 공간 데이터 처리를 위해 PostGIS를 도입하여 과거 위치 데이터를 효율적으로 저장하고 관리한다.

* **담당자**: 김지은
* **마감일**: 2025-12-26

### 안건 3: 헬스케어 정보 수집 및 PostgreSQL 공간 데이터 처리
* **논의 내용**: 반려동물 헬스케어 정보의 DB 스키마 설계 및 데이터 무결성을 위한 SQL 쿼리 점검 논의.
* **결정 사항**:
    * 헬스케어 정보를 수집하는 로직을 구현하고 전용 DB를 생성한다.
    * PostgreSQL 사용 시 발생할 수 있는 보안 위험이나 성능 저하 요인을 파악하여 SQL문을 수정한다.
* **담당자**: 양승준
* **마감일**: 2025-12-26

### 안건 4: 프론트엔드 사용자/펫 프로필 관리 기능 및 소셜 API 연동
* **논의 내용**: 사용자 및 반려동물 정보의 CRUD 작업에 대한 프론트엔드-백엔드 연동 완성도 제고 논의.
* **결정 사항**:
    * 사용자/펫 프로필 수정 및 상세 조회 API를 프론트엔드와 최종 연동한다.
    * 소셜 서비스의 API 연동 작업을 완료하고 사용자 피드백을 반영하여 UI를 수정한다.
* **담당자**: 이가은, 김현근
* **마감일**: 2025-12-26

### 안건 5: Helm Chart 수정 및 ArgoCD를 통한 마이크로서비스 배포
* **논의 내용**: 운영 환경 배포 안정성을 위해 기존 user-service의 설정을 보완하고 social-service의 지속적 배포 환경 구축 논의.
* **결정 사항**:
    * user-service의 Helm Chart를 수정하여 리소스 할당 등을 최적화한다.
    * social-service를 ArgoCD에 등록하여 배포 자동화 파이프라인을 완성한다.

* **담당자**: 손민정
* **마감일**: 2025-12-26
