## 회의 내용

### 안건 1: Kafka-Milvus RAG 파이프라인 및 성능 최적화
* **논의 내용**: Kafka로 수신된 다이어리 데이터를 임베딩하여 Milvus에 저장하는 자동화 과정에서 발생한 지연(Lag) 이슈 해결.
* **이슈 및 해결**:
    * **이슈**: Embedding 생성 작업의 부하로 인해 Kafka Consumer Lag이 증가하는 현상 발생.
    * **해결**: `@Async` 어노테이션을 적용하여 임베딩 생성 및 저장 로직을 비동기로 처리함으로써 처리 속도를 개선함.
* **구현 결과**:
    * **양승준**: 
        * `Diary Service`에서 일기 생성 시 `Healthcare Service`가 이벤트를 수신하여 자동으로 RAG 시스템에 반영하는 End-to-End 테스트 성공.
        * 데이터 흐름: Kafka Event 수신 → 텍스트 추출 → Titan Embedding 생성 → Milvus 저장.

### 안건 2: 다이어리 서비스 고도화 (MongoDB & AI)
* **논의 내용**: 비정형 데이터 처리를 위한 NoSQL 도입 및 생성형 AI 활용 기능 추가.
* **결정 사항**:
    * **김지은**: 
        * **MongoDB 도입**: 다이어리 사진의 비정형 메타데이터를 유연하게 저장하기 위해 MongoDB를 도입하고 연동을 마침.
        * **Spring AI 기능**: 사용자의 한 달 기록을 요약해주는 '월간 리캡(Monthly Recap)' 자동 생성 기능을 구현함.
        * 일기 생성 및 이미지 처리 프로세스 리팩토링 진행 중.

### 안건 3: 펫메이트 실시간 채팅 구현
* **논의 내용**: 펫메이트 매칭 후 사용자 간 소통을 위한 채팅 기능 개발.
* **진행 사항**:
    * **신재석**: [BE] WebSocket 기반의 채팅 메시지 송수신 및 세션 관리 로직 구현 중.
    * **김현근**: [FE] 채팅방 UI 컴포넌트 구현 및 백엔드 소켓 API 연동 병행 진행.

### 안건 4: 펫 관리 기능 및 헬스케어 인프라
* **논의 내용**: 펫 라이프사이클 관리 기능 연동 및 서비스 배포 준비.
* **진행 사항**:
    * **이가은**: 펫 삭제 및 '무지개 다리(추모)' 기능의 API 연동 작업 진행.
    * **손민정**: `healthcare-service`의 Dockerfile 및 GitHub Actions 워크플로우 작성, Pod 배포 진행 중.
