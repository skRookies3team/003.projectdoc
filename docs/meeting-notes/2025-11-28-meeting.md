## 회의 내용

### 안건 1: 핵심 알고리즘 및 고성능 처리 전략
- **논의 내용**:   
    - 마일리지 시스템의 정확성과 안정성을 보장하기 위한 알고리즘 설계 필요성 확인
    - 데이터 일관성 및 동시성 문제를 해결하기 위한 비동기 다중 쓰레드 처리 전략 논의 (RabbitMQ, Kafka 연동 방식 재검토).

- **결정 사항**: 
    - 비동기 처리는 Kafka를 활용한다.
- **담당자**: 양승준, 이가은, 김지은, 김현근, 신재석, 손민정
- **마감일**: 2025-12-01
### 안건 2: 프론트엔드 배포 전략 확정 (Next.js 이슈 해결)
- **논의 내용**: 
    - 문제 정의: Next.js (App Router, SSR 기반) 프로젝트를 정적 호스팅 서비스인 AWS S3에 배포할 경우, 서버 기능(Server Actions, Route Handlers 등)이 누락되어 애플리케이션이 정상 작동하지 않는 이슈 발생.
    - 해결 방안 논의: 1) Next.js의 output: 'export' 설정을 강제하고 SSR 안전 코드를 작성하거나 2) 전통적인 SPA 구조(React + Vite)로 변환하는 방안 검토.
- **결정 사항**: React + Vite 기반의 SPA 구조로 마이그레이션하여 S3 정적 배포를 확정하고, 빌드 및 배포 과정을 단순화한다.
- **담당자**: 양승준, 신재석
- **마감일**: 2025-12-01
### 안건 3: 기술/개발 환경 설정
- **논의 내용**:
    * DB를 PostgreSQL로 확정함에 따라 연동 작업 진행 및 ADR 도입 논의.
    * IntelliJ 내에서 코드 지원을 위한 AI CLI 도구 결정.
- **결정 사항**:
    * 모든 백엔드 서비스의 DB를 PostgreSQL로 확정하고 연동 작업을 진행한다.
    * IntelliJ 내 개발 생산성 향상을 위해 Gemini CLI를 사용하여 코드 생성 및 변환 작업을 진행하기로 결정한다.
- **담당자**: 양승준, 이가은, 김지은, 김현근, 신재석, 손민정
- **마감일**: 2025-12-01
### 안건 4: UI/UX 개선 사항 결정
- **논의 내용**:일기 생성 및 캘린더 관련 UI/UX 개선 논의.
- **결정 사항**:
    * 일기 생성 스타일 버튼 위치: 일기 생성 후 스타일 적용 버튼을 본문 입력 영역 하단으로 이동하여 사용 편의성을 높인다.
    * 캘린더 카드 넘김: 캘린더 대표 사진 클릭 시 카드 넘김 기능은 Nice-to-have 기능으로 분류하여 현재 스프린트에서는 제외하고 백로그로 이관한다.
- **담당자**: 양승준, 신재석
- **마감일**: 2025-12-01
