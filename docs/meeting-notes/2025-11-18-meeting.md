## 회의 내용

### 안건 1: 고객 관점 요구사항 정의서 전면 수정
- **논의 내용**:
  - 요구사항 정의서에서 사용된 기술적 언어가 고객이 이해할 수 있도록 바뀌어야 한다는 점 강조. 개발자의 관점이 아닌 사용자의 관점으로 재작성 필요
  - UI 설계 및 기술적 용어를 배제하고, 고객이 이해할 수 있는 방식으로 요구사항을 정리해야 함
  - ‘페이지’와 같은 개발자 중심의 용어는 사용하지 않도록 하고, 기능별로 도메인을 묶어서 정리할 것
  - 비즈니스적으로 중요한 기능은 우선순위를 높여 정의할 것
  - 핵심 기능을 20%로 뽑아서 회사의 비전과 일치하는 서비스로 구성
    
- **결정 사항**: 
  -	고객이 이해할 수 있도록 요구사항 정의서 전면 수정
  - 기술적 언어와 UI 설계 항목을 제외하고, 도메인별로 묶어 ‘what’ 중심으로 작성
  - 비즈니스 핵심 기능을 우선순위에 맞게 재정리
  - 요구사항 명세서와 설계서를 구분하여 작성
- **담당자**: 모든 팀원
- **마감일**: 2025-11-18

  - 프론트/백엔드 관점에서 기술적인 부분을 명확히 정의
- **결정 사항**

- **담당자**: 모든 팀원

### 안건 3: 코드 컨벤션
- **논의 내용**:
  - WebClient를 통한 다른 서비스 호출 클래스 설계
  - Kafka로 메시지를 처리하는 방법 및 메시지 전송에 사용될 클래스 설계
  - Controller/Service/Repository의 역할 분리 및 네이밍 규칙 설정
  - DTO 클래스 작성 방식 및 응답/요청 객체의 명확한 구분
  - Request와 Response를 구분하여 DTO 구조 설정 및 네이밍 규칙 명시
- **결정 사항**
  - WebClient:
    - PointClient와 같은 서비스 호출 클래스를 정의하여 WebClient 객체를 주입받아 사용
  - Kafka 메시지 처리:
    - Kafka 메시지는 Event로 전달되며, 이를 처리하는 BoardCreatedEvent와 같은 클래스를 사용, 메시지 소비자는 @KafkaListener를 이용하여 이벤트를 구독하고 처리
  - Controller/Service/Repository:
    - Controller, Service, Repository 역할을 명확히 구분하여 각 클래스의 책임을 분리
  - DTO 클래스 구조:
    - UserRequest와 UserResponse는 각각 사용자 요청과 응답을 처리하는 DTO로 구분, CreateUserDto와 같은 클래스는 생성자 패턴을 사용하여 요청 및 응답 객체를 구성, fromEntity 메소드로 엔티티 객체를 DTO로 변환하는 방식 정의
  - 네이밍 규칙:
    - DTO 클래스는 Request와 Response로 구분하고, 관련된 클래스는 CreateUserDto, CreateUserResponseDto 등의 형식으로 네이밍, CamelCase 네이밍 규칙을 준수하여 클래스 및 메소드 이름을 작성
- **담당자**: 모든 팀원
- **마감일**: 2025-11-18

### 안건 4: 수행계획서
- **논의 내용**:
  - 수행계획서에 따라 프로젝트 일정과 필요한 자원을 정리
  - 프로젝트 목적:
    - 반려인들이 여러 플랫폼을 효율적으로 사용할 수 있도록 통합된 서비스 제공
    - 사용자의 요구사항을 분석하고, 핵심 기능을 도출하여 MSA(Microservices Architecture) 기반으로 서비스 구조 설계
  - 기술 스택:
    - AI 기반 건강 관리, GPS 기반 위치 서비스, 커뮤니티 기능 등을 통합하여 사용자 맞춤형 서비스를 제공하는 것을 목표로 함
  - 프로젝트 실행 단계: 기획, 설계, 개발, 테스트, 배포의 단계로 진행
  - 사용자 인터페이스 및 경험:
    - 반려동물 관련 서비스를 사용자에게 쉽고 직관적으로 제공할 수 있는 UI/UX 설계
  - 인공지능(AI):
    - OpenAI GPT-4를 활용한 Vision-Text API, FastAPI, OpenCV 등을 사용해 이미지 및 텍스트 데이터 처리
  - 데이터 처리 및 모델 활용:
    - 데이터 분석과 관련된 AI 모델 및 챗봇 기능을 활용하여 사용자에게 맞춤형 피드백 제공
- **결정 사항**
  - MSA 기반 서비스 구성:
    - 각 서비스는 독립적인 마이크로서비스로 개발되며, RESTful API를 통해 다른 서비스와 통신하고 API Gateway를 통해 트래픽을 관리
  - AI 및 데이터 활용:
    - AI 기반 챗봇, 건강 관리 기능을 제공하고, GPS 및 IoT 기기를 통해 실시간 데이터를 수집하여 사용자 맞춤형 서비스 제공
    - AI는 건강 및 반려동물 관련 데이터를 분석하고 이를 통해 서비스 최적화
  - 사용자 인터페이스:
    - 반려동물 관련 쇼핑몰 및 GPS 기반 위치 찾기 기능 등 사용자 경험을 개선하는 UI/UX 설계
    - React와 Vite를 사용하여 프론트엔드 개발
  - 기술 스택:
    - Backend: Spring Boot, JPA, Redis, Kafka, API Gateway를 사용하여 MSA 구조로 서비스 설계
    - Frontend: React, Vite, TailwindCSS, Zustand 등 사용하여 반려인들을 위한 직관적인 사용자 인터페이스 구현
    - AI: OpenAI GPT-4를 이용한 Vision-Text API를 활용한 이미지 처리 및 텍스트 분석 기능
  - 배포 및 운영:
    - Docker, Kubernetes(K8s), EKS를 활용한 서비스 배포 및 관리
    - CI/CD 환경 구축을 통해 자동 배포 및 테스트
    - GitHub Actions, AWS EC2-RDS-S3 활용
