## 회의 내용

### 안건 1: Milvus Vector DB 통합 및 RAG 시스템 구축
* **논의 내용**: 챗봇의 답변 정확도를 높이기 위해 사용자 일기 데이터를 벡터화하여 저장하고, 이를 LLM 컨텍스트로 제공하는 RAG 아키텍처 구현.
* **이슈 및 해결**:
    * **이슈**: Milvus Collection 생성 초기화 실패 현상 발생.
    * **해결**: `autoCreateCollection` 옵션을 활성화하여 서버 기동 시 컬렉션이 자동 생성되도록 조치함.
* **구현 로직**:
    * **양승준**: 
        * `DiaryEmbeddingService` 구현: Diary 텍스트를 Vector로 변환하여 Milvus에 저장.
        * **RAG 로직**: 사용자 질문 → Vector 유사도 검색 → 관련 Diary 조회 → Claude에 컨텍스트로 제공.
        * 정확도 향상을 위해 **Top-3 유사 Diary**를 추출하여 답변 생성에 활용하도록 설계함.

### 안건 2: 펫메이트 실시간 채팅 기능 구현
* **논의 내용**: 매칭된 사용자 간의 실시간 소통을 위한 채팅 시스템 개발.
* **진행 사항**:
    * **신재석**: [BE] 펫메이트 매칭 서비스 내 WebSocket 기반 채팅 백엔드 로직 구현.
    * **김현근**: 
        * [FE] 채팅방 UI 레이아웃 및 메시지 전송 인터페이스 구현.
        * [BE] 채팅 관련 백엔드 서비스 로직 병행 개발.

### 안건 3: 다이어리 UI/UX 고도화 및 기능 확장
* **논의 내용**: 다이어리 서비스의 시각적 완성도 향상 및 외부 공유 기능 추가.
* **진행 사항**:
    * **김지은**: 
        * 다이어리 UI 개선 및 이미지 캐러셀(Carousel), 포토 갤러리 기능 적용.
        * 다이어리 보관함 및 AI 대시보드 데이터 연동.
        * 카카오톡 공유하기 API 연동을 통한 외부 공유 기능 구현.

### 안건 4: 인프라 파드 구성 및 알림 연동
* **논의 내용**: 서비스 안정화를 위한 인프라 작업 및 알림 기능 프론트엔드 연결.
* **진행 사항**:
    * **손민정**: `record-service` 관련 Milvus 및 Kafka 파드(Pod) 배포 및 서비스 간 통신 상태 점검.
    * **이가은**: 알림 내역 조회 및 알림 읽기 처리 API의 프론트엔드 연동 작업 진행.
