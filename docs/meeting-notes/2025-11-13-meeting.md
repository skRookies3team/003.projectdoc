## 회의 내용

### 안건 1: 와이어프레임 설계
- **논의 내용**:
  -	AI 일기 생성 기능 : AI 일기 생성 페이지에 접근하는 위치 설정 필요
  -	로그인 후 화면 : 로그인 후 홈 페이지 대신 대시보드 화면이 보여야 함
  - 반려동물 사진 애니메이션 : 대시보드에서 반려동물 사진을 보여주는 방식에 대한 고민
  -	보관함 페이지 : 보관함에 카테고리별로 파일을 분류할 수 있는 기능 필요
    
- **결정 사항**: 
  -	AI 일기 생성 기능 : 접근성이 좋은 위치에 일기 생성 기능을 배치하기로 결정, 상단 메뉴에 아이콘 또는 버튼으로 추가 예정
  -	로그인 후 화면 : 로그인 후 대시보드로 자동으로 이동하도록 설정, 기존 홈 페이지는 숨김 처리하고 대시보드 컴포넌트 위치를 조정하여 더 직관적인 화면으로 구성
  - 반려동물 사진 애니메이션 : 대시보드에서 반려동물 사진을 애니메이션 효과를 활용해 자연스럽게 표시. 또한, 반려동물의 일대기도 시각적으로 표현할 수 있는 공간 마련
  - 보관함 페이지 : AI가 자동으로 카테고리별 분류를 할 수 있도록 하고, 이를 반영한 UI를 설계
- **담당자**: 모든 팀원
- **마감일**: 2025-11-15

### 안건 2: 전체 기능 흐름 및 기능 정의
- **논의 내용**: 
  - 건강 데이터 분석 : 건강 데이터를 받아서 AI가 분석하는 기능 정의
  -	AI 페르소나 & 챗봇 : AI 챗봇 기능 추가 필요
  - 실종견 대시보드 : 실종견을 대시보드에서 보기 위해 어떻게 구성할지
  - 펫메이트 필터 기능 : 펫메이트 기능에 필터 추가 필요
  - 일기 캘린더 : 마이페이지에 일기 캘린더 추가 필요
  - 일대기 : 일대기를 대시보드에 표시
- **결정 사항**
  - 건강 데이터 분석 : 사용자가 기기를 통해 건강 데이터를 입력하면, AI가 분석 후 이를 대시보드에 실시간 차트 형태로 보여주도록 설정. 상세보기 기능도 추가하여 분석 결과를 구체적으로 확인할 수 있도록 함
  - AI 페르소나 & 챗봇 : AI 챗봇에 제미나이 API를 활용하여, 사용자가 AI와 자연스럽게 대화할 수 있는 기능을 추가. 페르소나 설정도 필요
  - 실종견 대시보드 : 대시보드에 실종견 정보를 표시하며, 사용자가 등록한 실종견 목록을 볼 수 있도록 함. 또한, 실종견 정보가 3개씩 랜덤으로 표시되도록 설정. 사용자가 목록을 모두 보기 위한 화살표 버튼 추가. 등록 버튼을 통해 사용자가 실종견을 등록할 수 있는 기능도 포함
  - 펫메이트 필터 기능 : 보호자 성별, 거리 필터만 선택 가능하도록 하고, 디폴트로 추천 기능을 활성화. 온/오프 기능을 추가하여 사용자가 메이트 기능을 항상 사용할 수 없도록 설정
  - 일기 캘린더 : 마이페이지에 일기 캘린더를 삽입하여 사용자가 생성한 일기를 쉽게 확인할 수 있도록 설정
  - 일대기 : 대시보드에 승준이의 일대기를 시각적으로 표현할 수 있는 공간 마련
- **담당자**: 모든 팀원
- **마감일**: 2025-11-15
